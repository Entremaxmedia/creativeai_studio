Improvement: when saving images, there should be a structured naming convention for them, as opposed to an open field text box. 

Full Naming Convention Structure
{category}-{product-id}-{type}-{variant}-{sequence}
Category Prefix (Required)
product - Product photography
adcreative - Ad images/graphics
element - Reusable design elements (badges, icons, textures)
lifestyle - Lifestyle/contextual shots
template - Design templates
Product/Campaign ID (When Applicable)
Use your SKU or internal product code
For ads, use platform abbreviation: fb, yt, rum
For campaigns: short campaign identifier
Type Descriptor (Required)
Products: null, photo, graphic, mockup, review
Ads: hero, variation, testimonial, comparison
Elements: badge, icon, texture, logo
Variant (Optional but Recommended)
Color, angle, setting, or specific differentiator
Keep to 1-2 words, hyphenated: front, black, outdoor, guarantee
Sequence Number (Required)
Always 2 digits: 01, 02, etc.
Dropdown Fields (Searchable with Add New option):
Category - Searchable dropdown with ability to add new categories
Product/Campaign ID - Searchable dropdown showing existing IDs with ability to add new
Type - Searchable dropdown (options change based on category selected) with ability to add new types
Sequence - Auto-incremented, but editable
Text Input Field: 5. Variant - Free-form text input (optional)
Each dropdown should:
Allow searching/filtering existing options as you type
Show "Add new: [your input]" option if what you're typing doesn't exist
Save new entries for future use, and ensure these are formatted correctly so it won’t break the image link
This keeps consistency while allowing flexibility to add new products, campaign IDs, or type descriptors as your business evolves.
